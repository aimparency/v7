# Verified Requirements

Verified against the official event overview and rules on July 18, 2026.
The official rules and event page remain the source of truth.

## Eligibility and Dates

- Individuals of legal majority in supported countries may enter.
- Germany is listed as an eligible country on the event page.
- Submission period: July 13, 2026 at 9:00 AM PT through July 21, 2026 at
  5:00 PM PT.
- Judging: July 22 through August 5, 2026.
- Winners are expected to be announced around August 12, 2026.

## Project

- Build a project using Codex and GPT-5.6.
- Enter one category: Apps for Your Life, Work and Productivity, Developer
  Tools, or Education.
- The project must install and run consistently as represented.
- Existing projects are eligible only when meaningfully extended using Codex
  and/or GPT-5.6 after the submission period began.
- Existing projects are judged only on work added during the submission period.
- Prior work and new work must be clearly distinguished, with evidence such as
  timestamped Codex sessions and dated commits.

## Submission

- Working project.
- Developer Tools category.
- English project description explaining features and operation.
- Public YouTube demo with audio, less than three minutes.
- Demo must show the project and explain how Codex and GPT-5.6 were used.
- Repository URL:
  - public with relevant licensing; or
  - private and shared with the two judging addresses specified in the rules.
- README must include setup instructions and explain:
  - collaboration with Codex;
  - where Codex accelerated the workflow;
  - key human product, engineering, and design decisions;
  - how GPT-5.6 and Codex contributed.
- `/feedback` Codex Session ID for the thread containing most core development.
- Because Aimparency is a developer tool:
  - installation instructions;
  - supported platforms;
  - a way to test without rebuilding from scratch, such as a hosted demo,
    sandbox, or test account.
- The working project must remain available free of charge and without
  restriction through judging.

## Judging

Stage one is a viability and theme-fit screen. Passing projects are evaluated
with four equally weighted criteria:

1. Technological implementation
2. Design
3. Potential impact
4. Quality and novelty of the idea

The Developer Tools prizes are USD 15,000 for first place and USD 10,000 for
second place. There is one winner at each level.

## Compliance Risks

- Do not present pre-July-13 functionality as Build Week work.
- Preserve dated commits and the primary Codex session.
- Do not claim GPT-5.6 API integration unless the product actually invokes that
  API. The defensible claim is that GPT-5.6 in Codex built the extension.
- Ensure all submission assets are owned or properly licensed.
- Do not use copyrighted music or unlicensed third-party material in the video.
