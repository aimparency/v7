# Storacha

## Challenge Information
- **Sponsor:** Storacha
- **Prize Pool:** $500 + Credits
- **Category:** Decentralized Storage

## Challenge Overview

Storacha challenges focus on building applications that leverage decentralized storage for AI agents, data persistence, and collaborative systems.

## Challenge Ideas

### 1. Persistent Agent Memory for OpenClaw Agents

**Problem:** OpenClaw agents lose all context when you restart or switch devices. Every conversation/preference starts from scratch.

**Solution:** Store agent state, conversation history, learned preferences on Storacha. Agent pulls its memory from Storacha on any device - laptop, phone, server.

Build agents that actually remember and evolve, not amnesia bots that reset daily.

### 2. Multi-Agent Coordination

**Problem:** Running multiple OpenClaw agents? They can't share knowledge or delegate tasks to each other. Each works in isolation.

**Solution:** Agents store shared task queues, execution logs, and knowledge on Storacha and you can share data using UCANs. Agent A learns something → stored on Storacha → Agent B instantly accesses it using UCANs.

Build agent swarms that collaborate and scale, not solo workers.

### 3. Decentralized RAG Knowledge Base

Build uncensorable, verifiable knowledge systems for autonomous agents.

**Problem:** AI agents need updated knowledge but centralized RAG systems are single points of failure. Data can be censored, lost, or tampered with.

**Storacha Solution:** Store training data, embeddings, and vector indices on Storacha. Content-addressed retrieval ensures data integrity. Any agent can query without centralized gatekeepers.

### 4. Storacha Tray Icon App

Build an app that sits in the system tray and allows users to quickly drag and drop files to upload them to Storacha.

### 5. Longest UCAN Delegation Chain

Build a web app that encourages users to create the longest UCAN delegation chain by sharing an initial delegation with their friends.

The app should:
- Generate an initial delegation
- Allow users to re-delegate to a friend
- Submit their delegation
- Store delegations on Storacha so the entire chain doesn't have to be passed around
- Show a leaderboard with the longest submitted chain
- Take steps to ensure users don't cheat (e.g., require unique verifiable drand randomness beacon for each delegation)

### 6. Drand Randomness Chain Backup

Build a system to backup to Storacha each new drand randomness beacon that is generated every 30s.

Requirements:
- The chain should be navigatable - each random value should link to the previous one
- The output should be consumable programmatically
- Consider using UCN to keep track of the head of the chain

### 7. Storacha Reseller

Build an app that re-sells Storacha space.

Requirements:
- Take a crypto payment for a set amount of storage space for a given time
- Create a space for a user
- Keep track of how much data is uploaded
- Issue delegation(s) to upload or proxy uploads to Storacha
- Remove user data when payment expires
- Bonus: allow top-ups to keep data around for longer
