# Existing Code

## Challenge Information
- **Sponsor:** Protocol Labs
- **Prize Pool:** $50,000
- **Category:** Meta Challenge (applies to continued projects)

## Challenge Overview

Build new solutions or expand existing codebases that meaningfully integrate partner bounties. Winners are eligible for the PL_Genesis Accelerator.

## Important Note: Fresh Code vs. Existing Code

**Fresh Code and Existing Code are not standalone challenges.**

You still need to build for one of the official sponsor challenges listed for the hackathon.

When you submit your project, you'll select whether your code is **Fresh** or **Existing**.

That selection determines which prize category your project is evaluated under.

**Don't forget to choose Fresh Code or Existing Code when you submit.**

## What is Existing Code?

Existing Code refers to projects that:
- Were started before the hackathon
- Have prior development history
- Are being expanded or improved during PL_Genesis
- May have participated in previous hackathons

## Eligibility

- Must demonstrate meaningful progress/expansion during the hackathon
- Must meaningfully integrate one or more partner bounties
- Must meet all requirements of the primary sponsor challenge(s) you're targeting

## Prizes

**Top 10:** $5,000 each (10 winners = $50,000 total)

## Additional Benefit

Winners are eligible for the **PL_Genesis Accelerator**.

## Judges

- Sabeen

## Submission Requirements

- Standard submission requirements apply based on the sponsor challenge(s) you're targeting
- Select "Existing Code" when submitting your project
- Clearly demonstrate what was built/improved during the hackathon period

## Judging Criteria

Judged according to the criteria of your primary sponsor challenge(s), with additional consideration for:
- Meaningful progress during the hackathon
- Quality of integration with partner technologies
- Demonstrated improvement over prior state
- Overall execution and completeness
